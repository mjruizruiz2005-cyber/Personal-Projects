# Contributing to Perfume Recommendation System

Thank you for your interest in contributing! This project welcomes contributions from both perfume enthusiasts and technical developers.

## 🎯 How You Can Contribute

### 🌸 **Perfume Domain Knowledge**
- Suggest new olfactory note classifications
- Propose additional archetype profiles
- Improve the compatibility matrix based on fragrance expertise
- Share insights on note harmonies from perfumery literature

### 💻 **Technical Contributions**
- Enhance the recommendation algorithm
- Add data visualizations
- Implement new features from the roadmap
- Improve code efficiency and documentation
- Write unit tests

### 📊 **Data Contributions**
- Expand the olfactory database with more notes
- Validate existing compatibility scores
- Contribute real-world perfume compositions for testing

## 🔧 Development Setup

1. **Fork the repository**
2. **Clone your fork**
   ```bash
   git clone https://github.com/yourusername/perfume-recommendation-system.git
   cd perfume-recommendation-system
   ```
3. **Create a virtual environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```
4. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

## 📝 Contribution Guidelines

### **Code Style**
- Follow PEP 8 for Python code
- Use descriptive variable names (e.g., `harmony_score` not `hs`)
- Add docstrings to functions explaining parameters and return values
- Keep functions focused on a single responsibility

### **Commit Messages**
- Use clear, descriptive commit messages
- Format: `[Category] Brief description`
- Examples:
  - `[Feature] Add seasonal recommendation logic`
  - `[Fix] Correct harmony score calculation for edge cases`
  - `[Data] Expand olfactory database with leather family notes`
  - `[Docs] Update README with new archetype examples`

### **Pull Request Process**
1. Create a new branch for your feature/fix
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes and commit them
3. Push to your fork
4. Open a Pull Request with:
   - Clear title describing the change
   - Description of what was changed and why
   - Screenshots/examples if applicable
   - Reference any related issues

### **Testing**
- Test your changes thoroughly
- If adding new features, include example usage
- Ensure the notebook runs end-to-end without errors

## 🌟 Priority Areas

Current priorities for contributions:

1. **Expanding the Olfactory Database**
   - Goal: 200+ notes covering all major families
   - Priority families: Leather, Fougère, Aromatic

2. **Improving Harmony Matrix**
   - Add empirical data from successful fragrances
   - Distinguish between "works well" vs. "iconic combination"

3. **Visualization Tools**
   - Perfume pyramid diagrams
   - Archetype similarity heatmaps
   - Composition balance charts

4. **User Interface**
   - Simple web interface for recommendations
   - Interactive archetype explorer

## 🚫 What We Don't Accept

- Binary files or large datasets without prior discussion
- Breaking changes without backward compatibility plan
- AI-generated code without human review and testing
- Changes that contradict core perfumery principles (e.g., putting base notes in top layer)

## ❓ Questions?

If you're unsure about something:
- Open an issue with your question
- Tag it with `question` label
- We're happy to provide guidance!

## 📜 Code of Conduct

- Be respectful and inclusive
- Welcome newcomers to both perfumery and programming
- Provide constructive feedback
- Assume good intentions
